# 贝叶斯可控性方案审核意见

审核身份：AAAI/AISI 审稿人视角  
审核对象：`贝叶斯todo.md`、Huys & Dayan 的 *A Bayesian formulation of behavioral control*、MinerU 提取内容、`report/0424report/scope_formalization_report.tex` 中的相关分析，以及当前 `digital_friction_mvp/proto` 实验框架。

## 1. 总体结论

`贝叶斯todo.md` 的当前方案是合理的，但它更适合被定位为：

```text
Bayesian-inspired controllability belief reconstruction
```

而不是完整的：

```text
Bayesian RL implementation of behavioral control
```

也就是说，它可以作为理论桥梁、post-hoc 分析指标和审计变量支撑论文叙事；但如果论文声称已经复现 Huys & Dayan 的完整 Bayesian RL 机制，审稿风险很高。

当前最稳的投稿策略是：

```text
Phase A + 理论形式化 + 审计图表
```

即：

- 不大改主机制。
- 用 Huys & Dayan 支撑 controllability belief 的理论动机。
- 用已有 attempt rows 和 `payload_json` 重建轻量 Beta belief。
- 证明高摩擦 / 低帮助世界会降低 control belief，并伴随 helplessness 与 avoidance 上升。

这条路线保守但可信，适合冲 AAAI/AISI；直接上复杂 Bayesian RL 反而可能引入更多机制耦合和消融负担。

## 2. 与 Huys & Dayan 的理论吻合度

Huys & Dayan 的核心观点不是“失败次数越多，所以越无助”，而是：

```text
个体从过去经历中学习到一个关于可控性的先验：
重要结果是否能通过自己的行为可靠控制？
```

论文中最重要的层次是第三层：

```text
controllably achievable reinforcement
```

也就是：个体真正关心的结果中，有多少可以由行为可靠获得或规避。

这一点与 `report/0424report/scope_formalization_report.tex` 的分析一致。报告明确区分了三层 control：

- outcome entropy
- controllably achievable outcomes
- controllably achievable reinforcement

并指出第三层更接近 helplessness 的心理实质：

```text
我在乎的结果，到底是不是受我行为控制？
```

因此，`贝叶斯todo.md` 把 digital friction 中的长期信念定义为：

```text
agent 对每个 digital task family 形成一个长期控制信念
```

这个理论转写是合理的。

尤其合理的是，todo 没有把 Bayesian control 直接等同于现有的 `event_level_uncontrollability`。后者是事件级主观不可控性；前者应该是跨经历累积出来的 task-family-level belief。二者有关系，但不是同一个变量。

## 3. 与当前实验框架的适配性

当前 `digital_friction_mvp/proto` 的主链是：

```text
任务分配 -> 策略选择 -> 结果生成 -> helplessness 更新 -> 经验记忆 / 审计落盘
```

这个框架非常适合先做 Phase A，因为 attempt rows 已经包含核心输入：

- `task_family`
- `strategy_type`
- `outcome_type`
- `event_level_uncontrollability`
- `support_mode`
- `avoid_reason`
- `task_appraisal`
- `helplessness_before / after / delta`
- `payload_json`

因此，Phase A 不需要改 agent 行为，只需要从现有日志重建：

```text
C_i,f,t ~ Beta(alpha_i,f,t, beta_i,f,t)
C_hat_i,f,t = alpha_i,f,t / (alpha_i,f,t + beta_i,f,t)
```

这一点符合 todo 中“所有新增指标必须能从 payload_json 或 attempt rows 中复现”的原则。

## 4. 快速数据 sanity check

我用现有 `exp_20260507_stream_memory_10d_60min` 的 12 个 world-run，按 todo 中的 z 映射做了一个快速重建。

平均 final `C_hat` 的排序如下：

| world | mean final C_hat |
|---|---:|
| `low_friction_high_assist` | 0.551 |
| `baseline_low_friction` | 0.391 |
| `high_friction_high_assist` | 0.301 |
| `high_friction_low_assist` | 0.211 |

这个排序符合理论预期：

- 低摩擦高帮助世界保留最高 control belief。
- 高摩擦低帮助世界最低。
- 高摩擦高帮助世界介于 baseline 和最差世界之间，说明帮助有保护作用，但无法完全抵消高摩擦。

按 run-level 聚合看，`mean_C_hat` 与关键指标方向也一致：

| 关系 | 快速相关方向 |
|---|---:|
| `C_hat` vs `helplessness_delta` | 强负相关 |
| `C_hat` vs `avoidance_delta` | 负相关 |
| `C_hat` vs `attempt_rate` | 正相关 |
| `C_hat` vs `success_rate` | 强正相关 |

这说明 Phase A 很可能可以产出清楚图表。

但必须注意：这些相关性不能直接当作独立因果证据。因为 `C_hat` 的输入来自 outcome / support / uncontrollability，而 helplessness 更新也使用了 outcome / uncontrollability / support mode 等变量。二者存在同源机制。审稿人会问：

```text
这个 Bayesian belief 是不是只是把已有 outcome 重新加权了一遍？
```

所以论文需要把它定位为“可审计的理论重构指标”，并补充 lagged prediction 或消融，才能增强因果说服力。

## 5. 当前 todo 的合理之处

### 5.1 边界意识正确

todo 明确写了：

- 不直接复刻原论文 action-outcome matrix。
- 不替换 `event_level_uncontrollability`。
- 不让 LLM 直接决定 Bayesian belief 数值。
- 第一阶段不让 Bayesian belief 影响 strategy、attribution、scope 或 helplessness 主公式。

这些限制是正确的。它避免了机制链过度膨胀，也避免论文被熟悉 computational psychiatry 的审稿人抓住“你没有真正实现 Huys & Dayan”的问题。

### 5.2 Beta belief 作为轻量代理合理

Beta 更新：

```text
alpha' = rho * alpha + w * z
beta'  = rho * beta  + w * (1 - z)
```

可以被解释为：

```text
把一次 digital task experience 转成可控 / 不可控证据，然后维护一个可审计的 task-family-level controllability belief
```

这虽然不是完整 Bayesian RL，但作为 belief audit 是合理的。

### 5.3 事件证据映射总体合理

todo 中的 z mapping 基本符合理论直觉：

- `success_self` 是强可控证据。
- `success_with_help + enabling_support` 是较强可控证据。
- `success_with_help + substituting_support` 较弱，因为 agency 被外包。
- 高 `event_level_uncontrollability` 的失败是强不可控证据。
- `risk_avoid` 和 `low_value_avoid` 不被强行解释为无助，是合理的。

这与 Huys & Dayan 对“重要结果是否可控”的关注是一致的。

## 6. 当前方案的主要审稿风险

### 6.1 风险一：不是完整 Bayesian RL

当前 todo 中的 Beta belief 没有维护：

```text
P(outcome | action, task_family)
```

也没有用 posterior predictive distribution 来计算 Q value 或选择 action。

因此，它不是严格的 Bayesian RL。更准确地说，它是：

```text
post-hoc Bayesian-style belief reconstruction
```

论文必须避免写成：

```text
we implement Huys and Dayan's Bayesian RL model
```

建议写成：

```text
Inspired by Bayesian formulations of behavioral control, we reconstruct a lightweight, auditable controllability belief over task families.
```

### 6.2 风险二：同源变量导致循环解释

`C_hat` 由 outcome / uncontrollability / support mode 更新；helplessness 也由 outcome / uncontrollability / support mode 更新。

所以如果只展示：

```text
C_hat 下降，与 helplessness 上升相关
```

审稿人会认为这只是机制重述。

需要补充：

- lagged prediction：`C_hat(t-1)` 是否预测下一次 avoid / help seeking / attempt。
- 控制变量：控制当前 helplessness、task_self_efficacy、world、agent fixed effect。
- 敏感性分析：不同 z mapping 下方向是否稳定。

### 6.3 风险三：avoid_without_attempt 证据较弱

没有尝试时，agent 没有真正观察到 action-outcome contingency。因此：

- `helpless_avoid` 可以作为低控制信念证据。
- `risk_avoid` 更像理性风险规避。
- `low_value_avoid` 更像任务价值低。

建议在分析里把 avoid 单独报告，甚至做两版：

```text
with avoid evidence
without avoid evidence
```

如果两版结论一致，会更有说服力。

### 6.4 风险四：z mapping 需要校准

当前 z mapping 是理论启发式，不是估计出来的参数。

投稿时至少需要：

- 报告它是 pre-specified mapping。
- 做 sensitivity analysis。
- 避免过度解释某个绝对数值。

建议重点解释排序和变化趋势，而不是强调 `C_hat = 0.211` 这样的绝对值。

### 6.5 风险五：seed 数偏少

当前 10-day stream-memory 实验有 3 个 paired seeds。方向很清楚，但统计显著性弱。

如果要把 Bayesian belief 作为主要结果，建议至少扩到：

```text
5-10 paired seeds
```

否则审稿人会认为结果只是 sanity check，而不是稳健实验。

## 7. 如果要更像 Bayesian RL，应该怎么改

如果目标不是“审计指标”，而是更像 Huys & Dayan 式 Bayesian RL，那么需要把 controllability belief 放进 action selection 之前，而不是只在事件之后重建。

### 7.1 把任务形式化为 contextual bandit / one-state MDP

每个 digital task episode 可以写成：

```text
context = agent profile + task_family + friction + support + recent memory
actions = attempt_self, seek_help_then_attempt, avoid
outcomes = success_self, success_with_help, failure_after_attempt, failure_even_with_help, abandon_midway, avoid_without_attempt
reward = task_value 成功收益 - risk/friction/失败成本 - agency loss
```

这不需要重写世界，只需要替换策略层的形式化。

### 7.2 用 Dirichlet posterior 表达 action-outcome belief

维护：

```text
P_i,f(o | a) ~ Dirichlet(alpha_i,f,a,o)
```

每次观察到 action `a` 和 outcome `o` 后更新：

```text
alpha_i,f,a,o' = rho * alpha_i,f,a,o' + 1[o = o']
```

预测分布：

```text
P_hat_i,f(o | a) = alpha_i,f,a,o / sum_o alpha_i,f,a,o
```

这比 Beta `C_hat` 更接近 Bayesian RL，因为它显式建模：

```text
action -> outcome
```

### 7.3 用 controllability prior 塑造 Dirichlet 先验

高 controllability prior 应该让不同 action 的 outcome distribution 更可区分：

```text
attempt_self -> success_self 更集中
seek_help_then_attempt -> success_with_help 更集中
avoid -> avoid_without_attempt 更集中
```

低 controllability prior 应该压平不同 action 之间的差异：

```text
不同 action 的 outcome 分布更相似
成功 / 失败更像由环境决定，而不是由 action 决定
```

这才更像 Huys & Dayan 所说的：

```text
低 control prior 会压平 predictive distribution，削弱动作价值差异，降低探索意愿。
```

### 7.4 用 posterior predictive 计算 Q value

用：

```text
Q(a) = sum_o P_hat(o | a, f) * R(o, task)
```

然后选择策略：

```text
policy(a) = softmax(Q(a) / temperature)
```

或者使用 Thompson sampling：

```text
1. 从 Dirichlet posterior 采样一个 action-outcome model
2. 对每个 action 计算 Q
3. 选择 Q 最大的 action
```

这样策略选择才真正由 Bayesian posterior 驱动。

### 7.5 加 exploration bonus

为了更像 Bayesian RL，还应加入探索价值：

```text
Q_Bayes(a) = expected_reward(a) + eta * uncertainty_bonus(a)
```

一个简单版本是：

```text
uncertainty_bonus(a) = 1 / sqrt(total_observations_for_action + 1)
```

低 control prior 可以降低 `eta` 或降低探索收益，从而复现“低 control 让世界不值得探索”的机制。

### 7.6 加 hierarchical generalization

为了贴近 learned helplessness，需要跨 task family 泛化：

```text
global controllability prior: chi_i
task-family controllability prior: chi_i,f
action-outcome posterior: P_i,f(o | a)
```

更新路径：

```text
一次 digital task episode
  -> 更新 P_i,f(o | a)
  -> 更新 chi_i,f
  -> 通过 task similarity 轻微更新相邻 family
  -> 轻微更新 global chi_i
```

这样才能解释：

```text
局部数字挫败如何变成跨任务的 digital helplessness。
```

## 8. 推荐执行路线

### 投稿冲刺版

建议只做：

```text
Phase A + 理论形式化 + 审计图表
```

必须补：

- `analysis_bayesian_control.py`
- world-level `C_hat` 曲线
- agent / task-family-level `C_hat`
- `C_hat` 与 helplessness / avoidance / attempt / success 的关系
- sensitivity analysis
- lagged prediction

论文中使用谨慎表述：

```text
We reconstruct a lightweight Bayesian-inspired controllability belief from auditable task-level evidence.
```

### 投稿增强版

如果时间允许，可以做 Phase B：

```text
proto_bayesian_control_memory
payload_json.auxiliary_audit.bayesian_control
```

但仍不影响策略和 helplessness 主公式。

这会增强可审计性，也更符合 AISI 对机制透明度的偏好。

### 完整 Bayesian RL 版

如果目标是后续强版本论文，可以新增：

```text
BayesianStrategyPolicy
```

它负责：

- Dirichlet action-outcome posterior
- posterior predictive Q
- Thompson sampling / softmax action selection
- controllability prior
- exploration bonus
- task-family generalization

然后做消融：

```text
full_bayesian_policy
no_bayesian_policy
no_controllability_prior
no_task_family_generalization
audit_only_beta_proxy
```

但这已经不是小修，而是机制层升级。

## 9. 审稿人最终意见

如果当前论文按 `贝叶斯todo.md` 的保守路线写，我的评价是：

```text
Weak Accept / Borderline Positive
```

理由：

- 理论动机清楚。
- 与 Huys & Dayan 的关键思想一致。
- 与当前 digital friction 实验框架适配。
- 指标可审计、可复现。
- 初步数据方向符合理论预期。

但我会要求作者澄清：

- 这不是完整 Bayesian RL。
- Beta belief 是 analysis proxy。
- 相关性不是独立因果证据。
- z mapping 需要敏感性分析。
- 需要 lagged prediction 或消融增强说服力。

如果论文声称已经实现完整 Huys & Dayan Bayesian RL，我的评价会变成：

```text
Weak Reject
```

理由：

- 没有显式 action-outcome posterior。
- 没有 posterior predictive Q。
- 没有 Bayesian exploration / value of information。
- 没有把 controllability prior 用于 action selection。
- 当前 Beta belief 更像审计指标，不是 RL policy。

因此，最建议的论文定位是：

```text
Huys & Dayan-inspired controllability belief audit for digital helplessness
```

而不是：

```text
full Bayesian RL model of digital helplessness
```

这个定位更诚实，也更容易通过审稿。
