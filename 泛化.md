# 两种泛化机制：Scope Spillover 与 Bayesian Controllability Generalization

## 1. 一句话区别

我们当前模型里至少有两种不同的“泛化”：

```text
Scope spillover:
  一次失败被 agent 解释成什么？
  这种解释会不会扩散到相似任务？

Bayesian controllability generalization:
  多次 action-outcome 经验说明了什么？
  agent 是否学到“类似数字任务整体上行动是否有用”？
```

更通俗地说：

```text
scope spillover = “我不行”的扩散
controllability generalization = “行动没用/行动有用”的扩散
```

这两者可以同时存在，但不能混成一个机制。

## 2. Scope Spillover 是什么

Scope spillover 是归因层面的泛化。它关心的是：

```text
一次失败之后，agent 怎么解释这次失败？
这个解释是只限于当前任务，还是扩散到相似任务？
```

例如：

```text
付款验证失败
→ agent 认为“我不擅长这类手机操作”
→ 相似的登录、认证、服务申请任务也受到影响
```

这里扩散的是：

```text
低自我效能
负性归因
任务范围感
“我这种人做不好”的解释
```

它主要影响的是 agent 对相似任务的信心、风险感和无助倾向，而不是直接从 action-outcome posterior 推断环境是否可控。

## 3. Scope Spillover 的文献支持

Abramson, Seligman, and Teasdale (1978) 的 reformulated learned helplessness 强调，人类无助的严重性、持久性和广泛性与归因维度有关，尤其是 internal / stable / global 这些维度。我们的 evidence matrix 中对应证据为：

```text
paper/mineruex/Learned Helplessness in Humans./auto/Learned Helplessness in Humans..md:7
证据片段: “stable or unstable, global or specific”
```

这支持我们把失败后的解释分成：

```text
stable vs unstable
global vs specific
internal vs external
```

也支持 scope spillover 作为归因范围的机制。

Tiggemann and Winefield (1978) 的情境相似性研究支持另一个关键点：泛化不是无限扩散，而是受 situation similarity 约束。我们的 evidence matrix 中对应证据为：

```text
paper/mineruex/Situation Similarity and the Generalization of Learned Helplessness - Tiggemann and Winefield 1978 (OCR copy)/auto/Situation Similarity and the Generalization of Learned Helplessness - Tiggemann and Winefield 1978 (OCR copy).md:17
证据片段: “situation similarity”
```

以及：

```text
paper/mineruex/Situation Similarity and the Generalization of Learned Helplessness - Tiggemann and Winefield 1978 (OCR copy)/auto/Situation Similarity and the Generalization of Learned Helplessness - Tiggemann and Winefield 1978 (OCR copy).md:111
证据片段: “similar ... dissimilar”
```

这支持我们当前的设计边界：

```text
失败不应该污染所有数字任务；
它更应该扩散到相似 task family。
```

所以 scope spillover 最稳的论文表述是：

```text
Scope spillover captures attributional generalization: when a negative digital episode is interpreted as stable or broad, its effect can transfer to similar task families, but the transfer is bounded by task similarity.
```

## 4. Bayesian Controllability Generalization 是什么

Bayesian controllability generalization 是行动-结果证据层面的泛化。它关心的不是“我怎么解释失败”，而是：

```text
从多次行动和结果看，
agent 的 action 是否真的能改变 outcome？
```

例如：

```text
付款任务:
  attempt_self -> failure
  seek_help_then_attempt -> failure

登录任务:
  attempt_self -> failure
  seek_help_then_attempt -> failure

材料上传:
  attempt_self -> failure
  seek_help_then_attempt -> failure
```

这时 agent 学到的不是单纯的：

```text
我不行
```

而是：

```text
类似数字任务里，不同行动好像也改变不了结果。
```

这是一种关于环境/任务域的可控性信念：

```text
这些数字任务整体上是否可控？
有价值结果是否能通过某个 action 稳定获得？
```

它应该来自 action-outcome posterior，而不是来自一次失败后的主观归因。

## 5. Controllability Generalization 的文献支持

Huys and Dayan (2009) 明确把 control 形式化为 action-outcome mapping 的性质，而不是简单成功率。他们讨论了 outcome entropy、achievable outcomes，以及 controllably achievable reinforcement。对应本地 MinerU 证据：

```text
paper/mineruex/A Bayesian formulation of behavioral control/auto/A Bayesian formulation of behavioral control.md:60
证据片段: “contingency, reliability or entropy”
```

这支持我们从以下指标派生 `C_family`：

```text
entropy_control:
  同一个 action 的 outcome 是否稳定？

action_contrast_control:
  不同 action 的 outcome distribution 是否真的不同？

reward_control_chi_lite:
  有价值 outcome 是否能被某个 action 稳定获得？
```

Huys and Dayan 还强调，有价值结果是否可控比单纯结果稳定更重要：

```text
paper/mineruex/A Bayesian formulation of behavioral control/auto/A Bayesian formulation of behavioral control.md:88
证据片段: “desirable outcomes”
```

这支持我们不要只看：

```text
action 是否稳定地产生某个 outcome
```

还要看：

```text
action 是否稳定地产生有价值 outcome，例如 success_self 或 success_with_help。
```

更关键的是，Huys and Dayan 专门讨论了可控性知识如何泛化到新环境：

```text
paper/mineruex/A Bayesian formulation of behavioral control/auto/A Bayesian formulation of behavioral control.md:92
证据片段: “taking actions and observing outcomes”
```

原文这一段的核心意思是：个体通过采取行动、观察结果来推断自己有多大控制力，并且一个关键问题是这种知识如何 generalize 或 transfer 到未来的新决策问题。

同一篇文章还说明，过去不可控环境会影响新环境中的 prior expectation：

```text
paper/mineruex/A Bayesian formulation of behavioral control/auto/A Bayesian formulation of behavioral control.md:50-54
核心含义: 动物在不可控经验后，会在新环境中预期结果仍然不可控，并因此减少控制尝试。
```

这正是 Bayesian controllability generalization 要建模的机制：

```text
旧任务中的 action-outcome evidence
→ 形成 controllability prior
→ 影响新任务/相似任务中的预测与策略
```

## 6. 两者最核心的不同

| 维度 | Scope Spillover | Bayesian Controllability Generalization |
|---|---|---|
| 泛化对象 | 归因、自我效能、无助感 | 可控性信念、action-outcome contingency |
| 关键问题 | 失败被解释成多广？ | 行动是否能改变结果？ |
| 主要证据 | event attribution、scope、similarity | P(outcome \| task_family, action) |
| 典型结果 | 相似任务信心下降 | 相似任务的 controllability prior 下降 |
| 理论文献 | Abramson et al. 1978; Tiggemann & Winefield 1978 | Huys & Dayan 2009; Seligman & Maier 1967 |
| 论文口径 | attributional generalization | action-outcome controllability generalization |

一个更直观的例子：

```text
付款验证失败
```

Scope spillover 会问：

```text
agent 是否把这次失败解释为“我不擅长这类数字任务”？
这个解释是否扩散到登录、认证、服务申请等相似任务？
```

Bayesian controllability generalization 会问：

```text
在付款、登录、上传等任务中，
attempt_self、seek_help_then_attempt、avoid 的结果分布是否有差别？
如果不同 action 的结果都差不多差，
agent 是否应该形成“类似数字任务低可控”的 prior？
```

## 7. 和当前模型的关系

当前模型已经有：

```text
scope_spillover
```

它主要处理归因层面的相似任务扩散。

当前 Huys-Dayan-lite 已经有：

```text
C_family
entropy_control
action_contrast_control
reward_control_chi_lite
```

它已经能回答：

```text
某个 task_family 内部，不同行动是否能稳定地产生有价值结果？
```

但当前还缺少更完整的：

```text
cross-family controllability belief generalization
```

也就是：

```text
从 payment / login / upload / service application 等相似任务中汇总 action-outcome evidence，
形成一个可迁移的 controllability prior，
再在新任务或低证据任务中作为弱先验使用。
```

这正是 `贝叶斯可控性信念上的泛化todo.md` 中建议新增的部分。

## 8. 为什么不能只用 Scope Spillover 替代 Controllability Generalization

如果只用 scope spillover，我们只能解释：

```text
agent 因为失败而信心下降，并把这种低信心扩散到相似任务。
```

但这解释不了另一种情况：

```text
agent 不是觉得“我个人不行”，
而是根据多次行动-结果经验发现：
这个数字系统里，我采取不同动作也很难改变结果。
```

这正是 learned helplessness 中 response-outcome noncontingency 的核心。Seligman and Maier (1967) 的经典实验强调，关键不是负面事件本身，而是结果与反应无关。对应本地证据：

```text
paper/mineruex/failure-to-escape-traumatic-shock-nnwn6t036k/auto/failure-to-escape-traumatic-shock-nnwn6t036k.md:63
证据片段: “independent of their responding”
```

所以我们需要两条路径：

```text
失败如何被解释
→ attribution / scope / self-efficacy

行动是否改变结果
→ action-outcome posterior / controllability belief
```

这两条路径都和 helplessness 有关，但不是同一个变量。

## 9. 论文中推荐写法

可以写：

```text
We distinguish attributional generalization from action-outcome controllability generalization. Scope spillover captures whether negative interpretations of a digital episode transfer to similar task families, whereas Bayesian controllability generalization summarizes action-conditioned outcome evidence into a transferable prior about whether valuable outcomes are behaviorally controllable in related digital tasks.
```

中文含义：

```text
我们区分归因层面的泛化和行动-结果可控性层面的泛化。
前者描述一次负性数字经历的解释是否扩散到相似任务；
后者描述 agent 是否从多个任务的行动-结果证据中学到“类似数字任务是否可控”的先验信念。
```

## 10. 不建议写法

不要写：

```text
Scope spillover 就是 Huys-Dayan controllability generalization。
```

原因：

```text
scope 是归因/自我效能扩散；
Huys-Dayan-inspired controllability 是 action-outcome evidence 的泛化。
```

不要写：

```text
低 controllability 一定导致 avoid。
```

更稳的写法是：

```text
低 controllability 会降低 action-value contrast，使有价值结果看起来不再稳定可达；在有支持可用时，它可能更合理地推动 seek_help_then_attempt，而不是直接推动 avoid。
```

不要写：

```text
我们完整实现了 Huys and Dayan 2009。
```

更稳的写法是：

```text
We implement a Huys-Dayan-inspired controllability diagnostic and propose a lightweight controllability-prior generalization mechanism.
```

## 11. 最终定位

最稳的机制定位是：

```text
Scope spillover:
  psychological attribution pathway

Bayesian controllability generalization:
  action-outcome evidence pathway
```

也可以画成：

```text
Digital friction episode
  ↓
Outcome + appraisal
  ├─ Attribution / scope pathway
  │    ↓
  │  self-efficacy and helplessness spillover to similar tasks
  │
  └─ Action-outcome evidence pathway
       ↓
     posterior P(outcome | family, action)
       ↓
     C_family / controllability prior
       ↓
     controllability belief generalization to related task families
```

这两个机制共同解释：

```text
为什么一次或多次数字摩擦不只是影响当前任务，
还会影响 agent 对相似数字任务的信心、控制感和后续行动策略。
```
