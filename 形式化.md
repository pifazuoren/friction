# 当前实验机制形式化

本文档形式化当前 `digital_friction_mvp` 实验机制。重点是已实现的 single-stage smoke / sanity test 链条，不把未来 staged design 或完整 multi-agent support ecology 写成已完成机制。

## 1. 基本对象

设 agent 集合为：

```text
i in I
```

实验时间按事件或天展开：

```text
t = 1, 2, ..., T
```

每个事件对应一个数字摩擦任务：

```text
e_t = (f_t, d_t, r_t, w)
```

其中：

```text
f_t in F
```

是 task family，例如 login、payment、form、search 等。

```text
d_t
```

是任务难度。

```text
r_t
```

是 friction type。

```text
w
```

是 world condition，例如 baseline low friction、high friction low assist 等。

行动集合为：

```text
A = {
  attempt_self,
  seek_help_then_attempt,
  avoid
}
```

outcome subtype 集合为：

```text
O = {
  success_self,
  success_with_help,
  failure_after_attempt_low_uncontrollability,
  failure_after_attempt_mid_uncontrollability,
  failure_after_attempt_high_uncontrollability,
  failure_even_with_help,
  abandon_midway,
  no_attempt,
  neutral_unknown
}
```

agent 的心理状态可以写成：

```text
s_{i,t} = (h_{i,t}, SE_{i,t,f}, trust_{i,t}, avoidance_{i,t}, M_{i,t})
```

其中：

```text
h_{i,t}
```

是 helplessness score。

```text
SE_{i,t,f}
```

是 agent 对 task family `f` 的 self-efficacy。

```text
M_{i,t}
```

是 memory，包括 status memory、stream memory、Bayesian posterior memory、controllability memory。

## 2. 单次事件主流程

一次事件的完整流程是：

```text
e_t
-> LLM Appraisal
-> pi_ref_t
-> pi_bayes_t
-> pi_final_t
-> a_t
-> y_t
-> psychology update
-> memory update
```

更具体地说：

```text
R_t = Retrieve(M_{i,t}, e_t)
```

```text
x_t = Appraise_LLM(e_t, s_{i,t}, R_t)
```

其中 `x_t` 包括 perceived difficulty、risk、felt control、expected help effectiveness 等。

然后 LLM semantic strategy 给出：

```text
pi_LLM_t(a)
```

旧 rule weights 记为：

```text
pi_rule_t(a)
```

但在当前 `semantic_v2` 中，`pi_rule_t` 不直接混入 `pi_ref_t`，主要保留为 context / audit / fallback。

## 3. LLM Reference Policy

当前 `semantic_v2` 的 reference policy 是：

```text
pi_ref_t = (1 - lambda_LLM) * pi_0 + lambda_LLM * pi_LLM_t
```

其中弱中性先验为：

```text
pi_0 = {
  attempt_self: 0.34,
  seek_help_then_attempt: 0.33,
  avoid: 0.33
}
```

如果 LLM 权重无效或 LLM confidence 低于阈值，则：

```text
pi_ref_t = pi_0
```

当前实验中：

```text
lambda_LLM = 0.50
```

所以可以理解为：

```text
pi_ref_t = 0.5 * pi_0 + 0.5 * pi_LLM_t
```

## 4. Bayesian Gated-Lite2: Action-Outcome Posterior

对每个 agent、task family、action，维护 Dirichlet posterior：

```text
alpha_{i,f,a,o}
```

其中：

```text
a in A
o in O
```

posterior predictive 为：

```text
P_t(o | f, a)
  = alpha_{i,f,a,o} / sum_{o' in O} alpha_{i,f,a,o'}
```

Bayesian 模块用 utility profile `U(o)` 把 outcome subtype 转成价值：

```text
Q_bayes_t(a)
  = sum_{o in O} P_t(o | f_t, a) * U(o)
```

然后用 softmax 得到 Bayesian policy：

```text
pi_bayes_t(a)
  = exp(Q_bayes_t(a) / tau)
    / sum_{a'} exp(Q_bayes_t(a') / tau)
```

其中 `U(o)` 当前是手写的 theory-informed utility profile，不是 human-calibrated utility。

## 5. Bayesian Gate

Bayesian 不是直接接管行动，而是通过 confidence gate 和 entropy gate。

每个 action 的 evidence confidence 是：

```text
conf_t(a) = n_{f,a} / (n_{f,a} + k)
```

其中 `n_{f,a}` 是该 task family 下该 action 的 update count。

posterior entropy 记为：

```text
H_t(a) = - sum_o P_t(o | f, a) log P_t(o | f, a)
```

gate 条件为：

```text
g_t(a) = 1[
  conf_t(a) >= theta_conf
  and
  H_t(a) <= theta_H
]
```

如果 gate 打开，则 Bayesian shift 为：

```text
Delta_bayes_t(a)
  = clip(
      conf_t(a) * (pi_bayes_t(a) - pi_ref_t(a)),
      -delta_max,
      delta_max
    )
```

如果 gate 关闭：

```text
Delta_bayes_t(a) = 0
```

Bayesian gated-lite2 的输出为：

```text
pi_tilde_t(a)
  = Normalize(pi_ref_t(a) + Delta_bayes_t(a))
```

再经过 probability floor 得到：

```text
pi_phase4_t
```

这就是 Phase4 Bayesian gated-lite2 的 final policy。

## 6. Huys-Dayan-Inspired Controllability

Bayesian gated-lite2 回答的是：

```text
某个 action 过去平均导致什么 outcome
```

Huys-Dayan-inspired controllability 回答的是：

```text
这个 task family 中，行动是否真的能可靠改变结果，并产生有价值 outcome
```

主控制行动集合为：

```text
A_C = {
  attempt_self,
  seek_help_then_attempt
}
```

当前默认不把 `avoid` 纳入主 `C_family`。

## 7. Entropy Control

对每个 control action，先只看 plausible outcomes：

```text
Omega_a subset O
```

然后重新归一化：

```text
Pbar_t(o | f, a)
  = P_t(o | f, a) / sum_{o' in Omega_a} P_t(o' | f, a)
```

entropy control 为：

```text
EC_t(a)
  = 1
    -
    (
      - sum_{o in Omega_a} Pbar_t(o | f, a) log Pbar_t(o | f, a)
    )
    / log |Omega_a|
```

family-level entropy control 为：

```text
EC_t(f)
  =
  sum_{a in A_C} conf_t(a) * EC_t(a)
  /
  sum_{a in A_C} conf_t(a)
```

如果证据不足，则回到 neutral：

```text
EC_t(f) = 0.5
```

## 8. Action Contrast Control

先把 outcome subtype 映射到 coarse bins：

```text
b(o) in {
  success,
  controllable_failure,
  uncontrollable_failure,
  dropout,
  no_task_evidence,
  unknown
}
```

coarse posterior 为：

```text
P_t(b | f, a)
  = sum_{o: b(o)=b} P_t(o | f, a)
```

两个 action 的 outcome distribution 差异用 TVD：

```text
TVD_t(a_i, a_j)
  = 0.5 * sum_b |P_t(b | f, a_i) - P_t(b | f, a_j)|
```

当前只有两个 control actions，所以：

```text
AC_t(f)
  = TVD_t(attempt_self, seek_help_then_attempt)
```

如果 balanced evidence 不足，则：

```text
AC_t(f) = 0.5
```

## 9. Reward Control / Chi-Lite

先把 utility 归一化到 `[0, 1]`：

```text
U_norm(o)
  = (U(o) - U_min) / (U_max - U_min)
```

每个 action 的 reward value 为：

```text
RV_t(a)
  = sum_o P_t(o | f, a) * U_norm(o)
```

reward achievability：

```text
RA_t(f)
  = max_{a in A_C} RV_t(a)
```

reward action gain：

```text
RG_t(f)
  = max_{a in A_C} RV_t(a)
    -
    min_{a in A_C} RV_t(a)
```

`chi_lite` 为：

```text
chi_t(f)
  = 0.70 * RA_t(f) + 0.30 * RG_t(f)
```

## 10. Family Confidence

balanced evidence 用 harmonic mean：

```text
n_eff
  =
  2 * n_attempt * n_help
  /
  (n_attempt + n_help + epsilon)
```

coverage 为：

```text
coverage
  =
  |{a in A_C: n_{f,a} >= n_min}|
  /
  |A_C|
```

family confidence 为：

```text
Conf_t(f)
  =
  coverage * n_eff / (n_eff + k)
```

这个设计防止某个 action 有很多数据、另一个 action 几乎没数据时，系统过早形成强 controllability belief。

## 11. C_family

raw controllability score 为：

```text
C_raw_t(f)
  =
  w_E * EC_t(f)
  + w_A * AC_t(f)
  + w_chi * chi_t(f)
```

当前默认：

```text
w_E = 0.25
w_A = 0.25
w_chi = 0.50
```

证据不足时做 shrinkage：

```text
C_t(f)
  =
  Conf_t(f) * C_raw_t(f)
  +
  (1 - Conf_t(f)) * 0.5
```

其中 `0.5` 是 neutral controllability belief。

## 12. Global Controllability Trace

after-event 更新 family diagnostic 后，如果：

```text
Conf_t(f) >= 0.50
```

则更新 global trace。

记：

```text
G_t
```

为 global controllability trace。

先做 decay：

```text
G_tilde_{t-1}
  = rho * G_{t-1} + (1 - rho) * 0.5
```

更新权重：

```text
w_t = eta * Conf_t(f)
```

然后：

```text
G_t
  =
  (1 - w_t) * G_tilde_{t-1}
  +
  w_t * C_t(f)
```

如果 family confidence 不足，则：

```text
G_t = G_{t-1}
```

## 13. Controllability-Aware Policy Modulation

Huys-Dayan-inspired controllability 接在 Bayesian gated-lite2 后面。

输入是：

```text
pi_phase4_t
```

如果 mode 不是 `gated_modulate`，则：

```text
pi_final_t = pi_phase4_t
```

如果：

```text
Conf_t(f) < theta_C
```

则 gate 关闭：

```text
pi_final_t = pi_phase4_t
```

如果：

```text
C_t(f) < T_low
```

执行 low-C flatten-only：

```text
gamma
  =
  delta_C_max
  * Conf_t(f)
  * (T_low - C_t(f)) / T_low
```

```text
pi_final_t
  =
  (1 - gamma) * pi_phase4_t
  +
  gamma * pi_uniform
```

直觉是：低可控性不直接推 `avoid`，只是降低 action preference contrast。

如果：

```text
C_t(f) > T_high
```

执行 high-C reduce-avoid：

```text
delta
  =
  delta_C_max
  * Conf_t(f)
  * (C_t(f) - T_high) / (1 - T_high)
```

令：

```text
a_star
  =
  argmax_{a in A \ {avoid}} Q_bayes_t(a)
```

或者使用 Phase4 policy 支持的最佳 non-avoid action。

然后：

```text
pi_final_t(avoid)
  =
  pi_phase4_t(avoid) - delta
```

```text
pi_final_t(a_star)
  =
  pi_phase4_t(a_star) + delta
```

其他 action 不变，再 normalize 并加 probability floor。

如果：

```text
T_low <= C_t(f) <= T_high
```

则：

```text
pi_final_t = pi_phase4_t
```

## 14. Action Sampling

最终行动从 final policy 中抽样：

```text
a_t ~ Categorical(pi_final_t)
```

当前重要边界是：

```text
pi_final_t
```

可以受 Bayesian gated-lite2 和 Huys-Dayan-lite 小幅影响，但 helplessness 不会被 Bayesian 或 `C_family` 直接改写。

## 15. Outcome Model

执行 action 后得到 outcome：

```text
y_t = T(e_t, a_t, s_{i,t}, w, epsilon_t)
```

其中 world condition `w` 影响 friction、support、accessibility 等环境参数。

outcome 包括：

```text
y_t = (
  outcome_type,
  support_mode,
  avoid_reason,
  u_t
)
```

其中：

```text
u_t
```

是 event-level uncontrollability。

然后 outcome 被映射成 Bayesian policy outcome subtype：

```text
o_t = classify(y_t)
```

例如：

```text
failure_after_attempt + u_t = high
-> failure_after_attempt_high_uncontrollability
```

## 16. Helplessness Update

helplessness 更新发生在 outcome 之后。

它的输入是：

```text
(
  h_t,
  outcome_type,
  consecutive_failures,
  support_quality,
  u_t,
  SE_t,
  felt_control,
  expected_help,
  avoid_reason,
  controllable_success_memory,
  support_mode
)
```

注意：这里没有直接输入：

```text
Q_bayes_t
pi_bayes_t
C_t(f)
G_t
```

如果 outcome 是 success：

```text
Delta h_t
  =
  base(outcome)
  -
  recovery(outcome, support_mode)
```

```text
h_{t+1}
  =
  clip(h_t + Delta h_t, 0, 100)
```

如果 outcome 是 failure / abandon / avoid：

```text
raw_t
  =
  base(outcome)
  +
  U_delta(u_t)
  +
  efficacy_loss(SE_t)
```

如果是 avoid：

```text
raw_t
  =
  raw_t * multiplier(avoid_reason)
```

如果过去有 controllable success memory：

```text
raw_t
  =
  raw_t * (1 - protection)
```

再加 damping：

```text
h_{t+1}
  =
  clip(h_t + damping(h_t) * raw_t, 0, 100)
```

因此当前机制链是：

```text
Bayesian / Controllability
-> action
-> outcome
-> helplessness
```

而不是：

```text
Bayesian / Controllability
-> helplessness
```

## 17. Bayesian Memory Update

outcome 后才更新 posterior。

对实际执行的 action `a_t`，更新：

```text
alpha_{i,f_t,a_t,o}
  <-
  rho * alpha_{i,f_t,a_t,o}
  +
  omega * 1[o = o_t]
```

如果 `rho = 1`，就是普通 Dirichlet count update。

update count：

```text
n_{f_t,a_t}
  <-
  n_{f_t,a_t} + 1
```

没有执行的 action 不更新。

这个更新只影响未来 event：

```text
M_{t+1} -> pi_{t+1}
```

不影响当前 action：

```text
a_t
```

## 18. Stream Memory Update

每次事件后写入 raw episode：

```text
m_t = description(e_t, a_t, y_t, u_t, h_{t+1})
```

写入：

```text
M_stream_{t+1}
  =
  M_stream_t union {m_t}
```

后续检索：

```text
R_{t+1}
  =
  Search(M_stream_{t+1}, query(e_{t+1}))
```

retrieved memory 进入 LLM appraisal / attribution / interview，但不直接作为 policy weight。

## 19. Scope Spillover

如果 outcome 是负面，且 attribution 正常，且 scope amplitude 超过阈值，则触发 scope spillover。

LLM 给出：

```text
a_scope_t in [0, 1]
```

event uncontrollability 为：

```text
u_t
```

任务相似度矩阵：

```text
S_{f,f'}
```

距离：

```text
d(f, f') = 1 - S_{f,f'}
```

Gaussian 权重：

```text
w_{f,f'}
  =
  exp(
    - d(f, f')^2 / (2 * sigma^2)
  )
```

归一化：

```text
what_{f,f'}
  =
  w_{f,f'} / sum_{k != f} w_{f,k}
```

扩散幅度：

```text
A_t = beta * u_t * a_scope_t
```

对其他 task family 的 self-efficacy 扣减：

```text
SE_{t+1,f'}
  =
  SE_{t,f'}
  -
  A_t * what_{f,f'}
```

这个机制表示：

```text
一次不可控失败
-> 相似任务自我效能下降
```

## 20. Audit Metrics

policy payload coverage：

```text
coverage_policy
  =
  # events with bayesian policy audit
  /
  # events
```

Huys payload coverage：

```text
coverage_Huys
  =
  # events with controllability audit
  /
  # events
```

intervention rate：

```text
IR
  =
  # events where pi_final_t != pi_ref_t
  /
  # events
```

TVD：

```text
TVD(pi, pi')
  =
  0.5 * sum_{a in A} |pi(a) - pi'(a)|
```

no post-outcome leakage 要求：

```text
pi_final_t independent of y_t given M_t, e_t, s_t
```

也就是说，当前 action policy 只能读取当前 event 之前的信息，不能读取当前 outcome。

## 21. 当前实验设置

当前 smoke run 是 single-stage steady setting：

```text
stage = steady
```

```text
T = 10 days
```

worlds：

```text
W = {
  baseline_low_friction,
  high_friction_low_assist,
  high_friction_high_assist,
  low_friction_high_assist
}
```

每个 world 有不同环境参数：

```text
w -> (friction, assist, accessibility, support)
```

当前关键配置：

```text
Bayesian mode = gated_lite
reference mode = semantic_v2
lambda_LLM = 0.50
delta_bayes_max = 0.10
Huys mode = gated_modulate
theta_C = 0.20
k_C = 4
delta_C_max = 0.10
```

## 22. 一句话总结

当前实验可以概括为：

```text
LLM appraisal
+ semantic reference policy
+ Bayesian action-outcome posterior
+ controllability-aware bounded modulation
+ post-outcome psychological state transition
+ stream/scope memory
```

其中最关键的因果顺序是：

```text
pre-outcome information
-> pi_final_t
-> a_t
-> y_t
-> h_{t+1}, M_{t+1}
```

而不是：

```text
y_t -> pi_final_t
```

也不是：

```text
C_t(f) -> h_{t+1}
```

所以当前机制的核心安全边界是：

```text
C_t(f) 只调节 action policy，不直接更新 helplessness
```
