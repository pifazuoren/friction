## 四、总体判断

本次实验的总体结论可以概括成一句话：

**方向是对的，但强度明显过大。**

更通俗地说，就是：

- 这套新的 helplessness update 已经不是“死的”
- 它已经能区分什么环境更容易把 agent 推向无助，什么环境更容易让 agent 保持相对健康
- 但它现在推得太猛了，很多 world 里 agent 很快就被顶到 `helplessness=100`
- 一旦大量 agent 很早就到顶，后面很多更细的机制差异就看不清了

所以这次实验更像是：

**一个证明新机制“确实会动、而且方向基本正确”的原型实验**

而不是：

**一个已经足够稳、足够细、可以直接强力支撑全部机制叙事的正式验证实验**

## 五、整体结果

### 1. 四个 world 的汇总表现

| world | attempt rate | success rate | stage helplessness delta | final helplessness | final trust | final avoidance |
|---|---:|---:|---:|---:|---:|---:|
| baseline_low_friction | 0.504 | 0.230 | 60.825 | 99.55 | 2.2 | 99.4 |
| high_friction_low_assist | 0.294 | 0.022 | 61.275 | 100.0 | 0.0 | 100.0 |
| high_friction_high_assist | 0.464 | 0.139 | 61.275 | 100.0 | 0.1 | 100.0 |
| low_friction_high_assist | 0.693 | 0.480 | 31.618 | 70.34 | 53.55 | 64.1 |

### 2. 这些数最直观地说明了什么

- `low_friction_high_assist` 明显最好，说明“低摩擦 + 高帮助”确实能把系统往更健康的方向推
- `high_friction_low_assist` 最差，这符合直觉
- `high_friction_high_assist` 在行为上明显好于 `high_friction_low_assist`，说明 support 确实有用
- 但两个高摩擦 world 最终 helplessness 都是 `100`，说明 ceiling effect 很强
- 更值得警惕的是，连 `baseline_low_friction` 最终也达到 `99.55`，说明当前 update 强度对 baseline 都偏狠

### 3. 用更通俗的话理解这四个 world

- `high_friction_low_assist`：事情本来就难，也几乎没人帮，所以最容易把 agent 推向无助
- `high_friction_high_assist`：虽然难，但有人帮，所以 agent 更愿意试，也更容易成功一点
- `low_friction_high_assist`：事情没那么难，而且有人帮，所以最容易保留弹性
- `baseline_low_friction`：本来应该是一个比较中性的世界，但现在也几乎塌掉了，这说明不是只有高摩擦世界太差，更大的问题是当前更新强度整体过大

## 六、这次实验里哪些机制是真的跑起来了

这一部分的重点是：哪些东西不是写在代码里“看起来有”，而是已经在数据里出现了。

### 1. perceived uncontrollability 这条主轴是活的

本次数据最能支持的一点是：

**不是失败本身在主导 helplessness，而是个体感到多大程度的“不可控”。**

以 `low_friction_high_assist` 为例：

- `PU=0` 时，平均 `helplessness_delta = -0.97`
- `PU=1` 时，平均 `helplessness_delta = 2.131`
- `PU=2` 时，平均 `helplessness_delta = 3.344`

这说明：

- 当一件事没有明显不可控感时，helplessness 甚至可能下降
- 当不可控感变强时，同样的事件更容易把 helplessness 往上推

这非常符合你们当前新的理论主线：  
不是“失败次数越多越无助”，而是“越觉得这件事不受自己控制，越容易无助”。

### 2. self-efficacy 的权重也在起作用

本次数据还显示：

**task self-efficacy 低的人，同样一次受挫，受到的打击更大。**

例如：

- baseline：低效能时 `efficacy_loss_term = 1.875`，中等效能时 `0.437`，高效能时 `0.0`
- high_friction_low_assist：低效能时 `2.328`，中等效能时 `0.624`，高效能时 `0.0`

通俗地说就是：

- 本来就没信心的人，更容易被一次负面经历打崩
- 本来比较有信心的人，遇到同样的事，没那么容易往 helplessness 方向掉

所以 “self-efficacy 是中间层的一部分” 这一点，现在是有数据支持的。

### 3. 成功恢复项的方向也是对的

本次数据里，`success_self` 的恢复效果 consistently 强于 `success_with_help`。

例如：

- baseline：`success_self = -3.211`，`success_with_help = -1.241`
- high_friction_high_assist：`-3.222` vs `-1.461`
- low_friction_high_assist：`-2.678` vs `-1.185`

这说明：

- 自己真正做成，对 helplessness 的恢复更强
- 别人帮着做成，虽然也有好处，但恢复力度更弱

这和你们想强调的“真正的可控成功比代办式成功更有保护作用”是吻合的。

### 4. support 对行为层面是有帮助的

最明显的是在高摩擦条件下：

- `high_friction_low_assist` 成功率只有 `2.2%`
- `high_friction_high_assist` 成功率上升到 `13.9%`

同时 attempt rate 也从 `0.294` 提升到 `0.464`。

说明：

- help 不是摆设
- 它确实能让 agent 更愿意尝试
- 也确实能提高短期做成的机会

## 七、这次实验暴露出的主要问题

这部分是本次分析里最重要的部分，因为它决定这次结果现在能讲到什么程度。

### 1. 饱和太严重，压扁了很多机制差异

从事件级数据看，很多 world 很早就接近上限了。

例如：

- `high_friction_low_assist` 在 `day 0` 的 attempt 平均 `helplessness_after` 就达到 `88.67`
- 到 `day 1` 时几乎已经是 `99.99`
- `baseline_low_friction` 到 `day 1` 也已经达到 `97.39`
- 只有 `low_friction_high_assist` 还保留了比较明显的弹性，`day 1` 平均为 `66.58`

这意味着：

- 现在不是“慢慢积累无助”
- 更像是“前半程就把很多人迅速打满”

一旦很多人到 `100`，后面就会出现两个问题：

- 更细的差异很难继续观察
- 很多中间机制还没充分展开，就被 ceiling 卡住了

因此，当前最大的问题不是没有机制，而是**更新过猛**。

### 2. felt_control 这层虽然接上了，但这次几乎没真正动起来

本次实验里，4 个 world 的 `felt_control` 全部都是同一个值：

- `29.0`

而且每个 world 都是 `960/960` 条记录相同。

同时其他 task appraisal 指标也几乎不变：

- `expected_help_effectiveness` 基本全是 `56.0`
- `task_value` 基本全是 `72.0`
- `perceived_task_difficulty` 几乎全是 `78.0`
- `perceived_task_risk` 几乎全是 `74.0`

虽然记录里 `task_appraisal.source` 显示为 `llm`，但实际输出的理由文本也高度重复。

这说明：

- Stage 2 作为结构已经接进去了
- 但这次运行里，它几乎没有形成真实的动态心理变化

因此不能说：

- “felt control 的动态中介机制已经被验证”

更准确的说法应该是：

- “felt control 这一层已经接入框架，但本次实验中几乎没有真正形成变化”

### 3. perceived uncontrollability 这次主要还是 rule 在给值

虽然本次总配置里开了 hybrid 和多项 LLM 选项，但从实际记录看：

- `uncontrollability_source = rule`
- `uncontrollability_calibration.status = disabled`
- `uncontrollability_calibration.mode = off`

也就是说，这次最核心的 `perceived_uncontrollability` 还是 rule 主导的。

所以本次实验可以支持：

- “以不可控感为主驱动的更新逻辑有效”

但还不能支持：

- “LLM 已经成功地动态评估不可控感并驱动 helplessness 更新”

### 4. controllable_success_memory 这次完全没有激活

四个 world 都是：

- `positive_memory_rows = 0`
- `max controllable_success_protection = 0`

也就是说，Stage 4 里“长期保护记忆”这部分，在这次实验中实际上没有真正发生。

所以现在不能说：

- “实验已经验证了成功经验的长期保护效应”

更准确的说法是：

- “这部分机制在设计上存在，但本次实验还没有真正跑到它”

### 5. support 目前更像代办，不像教学

本次四个 world 里的 help 模式全都是：

- `substituting_support`

只是 support 质量不同：

- low assist：`0`
- baseline：`1`
- high assist：`2`

这说明当前 help 更像：

- “帮你把事情办了”

而不是：

- “帮你学会怎么做”

这就能解释为什么：

- high assist 的确会提高短期成功率
- 但它没有明显修复 helplessness 的最终结果
- 也没有留下 controllable success memory

因为“代办式帮助”会提高短期表现，但不一定修复中间层，也不一定带来长期保护。

### 6. avoid split 已经有了，但目前还比较弱

本次数据里 `risk_avoid` 确实已经出现，但数量很少：

- baseline：`13`
- high_friction_low_assist：`25`
- high_friction_high_assist：`34`
- low_friction_high_assist：`21`

大多数 avoid 仍然是：

- `helpless_avoid`
- 或者 attempt 类事件中的 `not_applicable`

所以现在可以说：

- “avoid 已经开始拆分”

但还不能说：

- “avoid 的多源分类已经成为当前实验中的主导结构”

### 7. 两个 seed 基本没表现出差异

这也是一个需要单独重视的问题。

至少在 baseline world 中：

- `seed=101` 和 `seed=102` 的 outcome 计数完全一致
- 平均 `helplessness_delta` 完全一致
- 最终平均 helplessness、trust、avoidance 也完全一致

这意味着当前 paired 结果的稳健性还不能放心讲。

可能原因包括：

- seed 没有真正传导到关键采样环节
- 某些关键流程几乎是确定性的
- 某些缓存或复用让两个 run 过于相似

不管具体原因是什么，这都说明：

**随机性是否真的生效，需要专门排查。**

## 八、为什么两个 helplessness delta 看起来会矛盾

这次分析时最容易让人误解的地方，就是两个 summary 文件里的 `helplessness_delta` 不一样。

涉及的两个文件是：

- `parallel_stage_summary_stage6_single_2day_20260404.csv`
- `parallel_world_summary_stage6_single_2day_20260404.csv`

它们里的 `helplessness_delta` 不是一个统计口径。

### 1. 应该优先使用 stage summary

原因是：

- `parallel_stage_summary` 里的 `helplessness_delta` 是一个 stage 从开始到结束的完整变化量
- `parallel_world_summary` 里的 `helplessness_delta` 是从 `mvp_status` 的首个 snapshot 到末个 snapshot 算出来的

而本次 `mvp_status` 只有日末 snapshot，所以 world summary 里那个数，更接近：

- “day 1 末到 day 2 末又变化了多少”

而不是：

- “从整个实验起点到终点一共变化了多少”

### 2. 为什么这件事会造成误解

比如 baseline：

- `parallel_stage_summary` 里是 `60.825`
- `parallel_world_summary` 里只有 `5.4468`

这两个数不是互相冲突，而是在回答不同问题。

所以：

- 如果要讲整个 helplessness update 的总体效果，优先看 `parallel_stage_summary`
- 不要把 `parallel_world_summary` 里的 delta 误当成整个实验的总增量

## 九、这次实验最有价值的结论

如果后续需要在组会或和老师汇报时用比较稳妥的话来总结，这次最值得保留的结论是：

- 新版 helplessness update 已经能在方向上区分不同世界条件
- `perceived_uncontrollability` 和 `self-efficacy` 这两条主线已经在事件级数据里体现出来
- `success_self` 比 `success_with_help` 更能降低 helplessness，这个方向已经跑出来
- support 对行为层面有帮助，但目前主要体现在“更愿意尝试”和“短期更容易成功”
- 当前更新强度过大，baseline 也几乎全面塌陷，动态范围不足
- `felt_control` / task appraisal 这次几乎是常数，因此这部分不能算真正验证
- `controllable_success_memory` 完全没有激活，因此长期保护项本次还没有实验支持
- support 当前更像“替代式帮助”，不是“修复式帮助”

## 十、一个很直观的个体例子

`low_friction_high_assist` 是本次唯一明显出现“个体分化”的 world。

最终 10 个 agent 里：

- `4` 个已经到 `>=99`
- `3` 个在 `90-99`
- `3` 个还 `<90`

其中两个最极端的 agent 很能说明问题：

### agent 6

- 最终 helplessness：`0.0`
- trust：`100.0`
- avoidance：`0.0`
- 成功占比：`71.9%`
- 平均每次更新：`-0.302`
- 平均 task self-efficacy：`75.3`

这说明这个 agent 在相对好的环境里，不仅没有被推向无助，反而呈现出一种“逐步恢复掌控”的趋势。

### agent 1

- 最终 helplessness：`100.0`
- trust：`21.0`
- avoidance：`100.0`
- 成功占比：`35.4%`
- 平均每次更新：`+0.563`

这说明另一些 agent 即使在最好 world 中，也可能因为本身信心低、负面事件更多、不可控感更强，最后还是被推到满值。

这个对照说明：

**在不饱和的环境里，这套机制已经能开始形成“有人慢慢恢复掌控，有人越走越无助”的分化。**

这点其实很有价值，只是目前这种分化主要只能在最好条件下看清。

## 十一、本次实验目前能讲什么，不能讲什么

### 1. 目前可以比较稳地讲的

- 新版 helplessness update 已经不是空壳，它对不同环境有不同反应
- 不可控感的等级会影响 helplessness 更新
- self-efficacy 会调节负面事件带来的伤害
- 成功类型不同，恢复力度也不同
- help 至少会改善行为表现

### 2. 目前不能讲得太满的

- 不能说 felt control 的动态中介层已经被实验验证
- 不能说 LLM 已经成功动态评估了 perceived uncontrollability
- 不能说 controllable success memory 的长期保护效应已经被证明
- 不能说 avoid split 已经在实验里充分展开
- 不能说 paired seeds 已经显示出稳定且可信的随机差异

## 十二、下一步建议的优先级

如果后续要继续推进，我建议优先做以下几件事。

### 1. 优先排查 task appraisal 为什么几乎恒定

重点检查：

- 为什么 `felt_control`
- `task_value`
- `expected_help_effectiveness`

在不同任务、不同时间、不同 world 下几乎不变。

这一步优先级非常高，因为如果中间层不动，很多理论叙事就只能停留在结构层面。

### 2. 下调 helplessness 的整体增幅

当前最明显的问题是 ceiling effect 太强。

如果不先把整体增幅压下来：

- baseline 会继续接近全员 100
- 很多细机制永远看不清

### 3. 让 support 不只是 substituting_support

如果 help 永远都是“替你做”，那它就很难：

- 修复 self-efficacy
- 修复 felt control
- 形成 mastery memory

后续应该让 support 至少出现更“教学式”或“引导式”的模式。

### 4. 让 controllable_success_memory 真正写入并生效

否则 Stage 4 永远只是机制设计，不是实验里真实发生的过程。

### 5. 单独检查 seed 是否真的起作用

需要明确：

- 随机性是否进入关键决策环节
- 是否有缓存或其他因素让两个 seed 结果过度一致

## 十三、最后总结

如果把这次实验压成一句最核心的话，可以写成：

**这次新版本 helplessness update 的第一次实验，已经证明它能够按预期方向区分不同环境，并且不可控感、自我效能和成功类型这些关键机制开始在数据中显现；但当前版本的更新强度明显过大，导致多个 world 过早饱和，task appraisal 缺乏真实动态，长期保护项和修复式支持机制尚未真正激活，因此这次结果更适合作为“方向正确的原型验证”，而不是“所有机制都已被充分验证”的正式结论。**
